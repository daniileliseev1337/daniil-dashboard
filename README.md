# Личный рабочий центр — ДАНИИЛ

Веб-приложение для ведения проектов и финансов, подключённое к облачной БД Supabase.

## Что внутри

Это полноценное React-приложение, собираемое через Vite. Оно работает как
обычный сайт — открывается по своему адресу, доступно с любого устройства,
синхронизирует данные через Supabase. В нём четыре раздела: Дашборд,
Проекты, Финансы, Аналитика. Поддерживается импорт банковских выписок
(CSV из Тинькофф/Сбер/Альфа/Яндекс и PDF из Яндекс Банка), автокатегоризация
транзакций, экспорт отчётов через печать в PDF.

## Что нужно сделать перед запуском

1. **Аккаунт GitHub** (бесплатный) — для хранения исходников
2. **Аккаунт Vercel** (бесплатный) — для размещения сайта
3. **Аккаунт Supabase** (уже есть) — для хранения данных

База данных Supabase уже настроена и работает, твой админский профиль
тоже создан. Эти шаги делать заново не нужно.

## Способ 1: через веб-интерфейс GitHub (без компьютера)

Если у тебя нет под рукой компьютера и ты работаешь с телефона.

1. Зайди на github.com, авторизуйся
2. Нажми "+" в правом верхнем углу → New repository
3. Имя: `daniil-dashboard`
4. Выбери Private (приватный, чтобы код не был публичным)
5. Не ставь галочку "Add README" — она у нас уже есть
6. Жми Create repository
7. На следующей странице найди ссылку "uploading an existing file"
8. Перетащи туда все файлы из распакованного архива
9. Внизу страницы напиши Commit message и нажми Commit changes

После этого в репозитории должны появиться все десять файлов и папка `src/`.

## Деплой на Vercel

После того как код в GitHub:

1. Зайди на vercel.com, авторизуйся через GitHub
2. New Project → выбери репозиторий `daniil-dashboard`
3. Vercel сам определит что это Vite-проект, ничего настраивать не нужно
4. **ВАЖНО:** перед деплоем разверни секцию Environment Variables и добавь:

```
VITE_SUPABASE_URL = https://pzdzyaswjlqiifmacygr.supabase.co
VITE_SUPABASE_KEY = sb_publishable_5LwhLNwSKaoxBYmfxUCHbA_0kB5KPJq
```

5. Нажми Deploy
6. Через минуту-две сайт будет на адресе типа `daniil-dashboard.vercel.app`
7. Открой его, войди с email и паролем, проверь что всё работает

## Как добавить иконку на iPhone

1. Открой свой сайт на vercel.app в Safari (не Opera, именно Safari)
2. Нажми кнопку "Поделиться" (квадратик со стрелкой вверх) внизу экрана
3. В меню выбери "На экран «Домой»"
4. Можно поменять название на короткое, например "Даниил"
5. Иконка появится на главном экране телефона

После этого сайт открывается как обычное приложение — без адресной строки,
во весь экран. С функциональной точки зрения это полноценный замены нативного
приложения, и его можно будет использовать как ежедневный инструмент.

## Локальная разработка

Если захочешь тестировать изменения у себя на компьютере перед деплоем:

```bash
# Установка зависимостей (один раз)
npm install

# Скопировать .env.example в .env (один раз)
cp .env.example .env

# Запуск локального сервера разработки
npm run dev
```

Сайт откроется по адресу `http://localhost:5173`. Все изменения в коде
применяются мгновенно без перезагрузки страницы.

## Если что-то не работает

Самые частые проблемы при первом деплое:

- **Build failed** на Vercel с упоминанием Tailwind или PostCSS — проверь
  что файлы `tailwind.config.js` и `postcss.config.js` залиты в репозиторий
- **Build failed** с ошибкой про переменные окружения — значит
  Environment Variables не настроены в проекте Vercel
- **Сайт открывается, но "Load failed" при входе** — значит ключ
  Supabase неверный, проверь Environment Variables ещё раз
- **Сайт открывается, видно экран входа, но «Аккаунт ожидает одобрения»** —
  это означает что email-confirmation в Supabase не пройдено; зайди в
  Supabase Auth Users, найди свой email и нажми "Confirm email" вручную
