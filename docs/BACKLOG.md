# Backlog разработки — klimat-pro-AI

Задачи на будущее, записаны по ходу работы (не начаты). Перед реализацией — brainstorm → спек → план.

## 1. Коммуникации — мессенджер ИЛИ объявления (развилка)
Два пути, выбрать один:
- **Вариант A — полноценный мессенджер:** выбор адресата (личные сообщения) + групповой чат. Тяжёлый: realtime-канал, RLS-приватность переписки, UI чата, уведомления (push/inbox), вложения.
- **Вариант B — объявления/новости (если чат тяжело):** новая карточка на **дашборде** — «Объявления» / «Важно» (название продумать). Любой пользователь пишет общую информацию, **не привязанную к проектам**. Аудитория: выбранные пользователи ИЛИ все. Опционально — привязка к проекту («касается проекта X»).
- Решение по пути — за владельцем. Объявления заметно проще мессенджера и закрывают основную потребность «общая инфа для всех».

## 2. Темизация — выбор дизайна пользователем (очень далёкое будущее)
Несколько вариантов оформления + переключатель темы (в дополнение к уже сделанному режиму «Высокий контраст»). Хранение — per-device (localStorage) или в профиле.

## 3. Открытые вопросы дизайна (текущие)
- **Android-тусклость:** проверить на устройстве сотрудника поддержку `mask-composite` (золотые рамки карточек) и `backdrop-filter` (frosted-стекло). Если рамки не рисуются — нужен CSS-фолбэк (обычная граница), контраст это не лечит.
- **Высокий контраст → возможно сделать дефолтом** (владельцу понравилось на iPhone/ПК). Уточнить ощущение «размытости» (вероятно `backdrop-filter: blur` на шапке/стеклянных карточках).
- **Живой фон** — реакция на курсор (десктоп) и скролл/касание (мобайл). [в работе]

## 4. Онбординг / навигационный гид по возможностям
Многое уже реализовано (Дашборд, Проекты, Финансы, Аналитика, Задачи, импорт выписок,
права/команда, клиентская база, PWA), но новому пользователю — и владельцу как напоминание —
негде увидеть «что тут есть и где». Нужна навигация-инструкция. Варианты:
- **Вариант A — онбординг-тур:** пошаговые подсказки (coachmarks) при первом входе, подсвечивают
  разделы; повторный запуск из «Профиль/Помощь». Самый наглядный, но больше всего нового UI.
- **Вариант B — статический раздел «Возможности/Помощь»:** страница/модалка со списком разделов
  и что каждый делает. Дешевле тура, проще поддерживать, но пассивный (надо открыть самому).
- **Вариант C — командная палитра как навигатор (переиспользует готовое):** в проекте уже есть
  `src/components/CommandPalette.jsx` — расширить её в точку входа: поиск по разделам/действиям +
  краткие описания. Минимум нового кода, работает с клавиатуры. Хорошо комбинируется с B.
- Хранение «тур пройден» / «подсказки видел» — per-device (localStorage) или в профиле.
- Решение по варианту — за владельцем. Перед реализацией: brainstorm → спек → план.
