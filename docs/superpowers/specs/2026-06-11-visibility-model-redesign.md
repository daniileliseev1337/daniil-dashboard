# Переработка модели видимости проектов + команда при создании (№7)

Дата: 2026-06-11. Согласовано с владельцем (ревью спека пропущено по его указанию).

## Проблема
Текущая RLS `projects` SELECT даёт: владелец / админ(всё) / член команды (ЛЮБОЙ видимости) /
team+одобрен / marketplace+одобрен / selected(мёртвый). Следствия, которые не нравятся владельцу:
- «Командный» виден ВСЕМ одобренным (а не только назначенной команде).
- Член команды видит проект любой видимости, включая «Личный».

## Новая модель видимости
- **Личный (private):** видит только владелец (+ админ тайно).
- **Командный (team):** видит владелец + назначенная команда (`project_members`).
- **Маркетплейс (marketplace):** видят все одобренные (единственная «публичная» категория).

### Новая RLS `projects_select`
```sql
owner_id = auth.uid()
OR is_admin()
OR (visibility = 'team'        AND is_project_member(id))
OR (visibility = 'marketplace' AND is_approved())
```
Убрано: безусловный `is_project_member(id)`, ветка `team AND is_approved()`, ветка `selected`.
INSERT/UPDATE/DELETE-политики не меняются.

## Админ (вариант A — выбор владельца)
RLS для админа НЕ меняем — `is_admin()` оставляет ему технический доступ ко всему (тайно).
В UI вкладки «Проекты»:
- по умолчанию у админа скрыты ЧУЖИЕ ЛИЧНЫЕ проекты (`ownerId != me && visibility = 'private'`);
- чип-переключатель **«Все проекты (админ)»** (виден только админу) снимает фильтр → показывает всё.
Обычные пользователи фильтруются самой RLS, клиентский фильтр их не касается.

## №7 — команда при создании (альтернатива: секция команды видна ВСЕГДА)
- Секция «Команда проекта» в `ProjectForm` показывается всегда (и при создании, и при правке).
- **Создание** (нет `initial.id`): команда собирается в локальный черновик `teamDraft`
  (autocomplete `search_approved_users` + роль viewer/editor на каждого, паттерн как у долей/исполнителей).
  После `insertProject` — `addProjectMember` для каждого участника черновика.
- **Правка** (есть `initial.id`): существующий `MembersManager` (пишет в БД сразу), без изменений.
- **Подсказка:** когда `visibility != 'team'`, под секцией команды — заметка
  «Участники увидят проект только при видимости „Командный“» (т.к. RLS даёт доступ команде лишь для team).

## Миграция / совместимость
- Существующие `team`-проекты станут видны только команде; у кого команда пуста — только владельцу
  (владелец назначит команды вручную).
- Старые `project_members` у личных проектов перестанут давать видимость (это и есть цель).
- `selected` (мёртвый режим, `project_visibility` пуст) убирается из RLS.

## Тестирование
- RLS-проверка на живой БД: обычный юзер НЕ видит чужой личный и чужой командный (где не член);
  видит свой + командный где член + маркетплейс. Админ видит всё (через is_admin()).
- Клиент: админ по умолчанию без чужих личных; чип «Все проекты» их показывает.
- Создание проекта с командой → участники появляются в `project_members`.
