# Дизайн: интерактивные карточки проекта (quick-edit) + реорганизация формы во вкладки

**Дата:** 2026-06-19
**Замечание:** #7 (интерактивные карточки) + сопутствующая разгрузка перегруженной ProjectForm.
**Файл:** `src/App.jsx` — `Projects` (карточки), `ProjectForm`, `MembersManager`.
**Источник дизайна:** brainstorming + Workflow judge-panel (3 концепции → синтез), одобрено владельцем.

## Проблема

1. Чтобы сделать частое действие по проекту (сменить стадию, отметить платёж, подвинуть
   дедлайн, назначить исполнителя, поправить команду), приходится открывать тяжёлую форму.
2. Сама `ProjectForm` (модалка) перегружена ~16 блоками сплошным списком — пользователь
   путается, что и куда нажимать.

## Цель

- **Quick-edit прямо на карточке** для 5 частых действий, без захода в форму.
- **Реорганизовать форму во вкладки** (4 вкладки) для ясности.
- Кнопка-карандаш (полная форма) остаётся.

## Часть A — Quick-edit на карточке

### Единый компонент `QuickEdit`
Один presentational-компонент с пропом `mode` (`stage | payment | deadline | executor | team`).
Рендерится через **React Portal в `document.body`**, позиционируется `position:fixed` по
координатам триггера (`getBoundingClientRect`). Portal+fixed выбран сознательно — снимает три
бага, которые были бы у `position:absolute` внутри карточки: обрезку родительским `overflow`,
уезд за правый край экрана (зеркальный сдвиг по rect), закрытие/смещение при скролле.

- **Один глобальный state** `activeQuickEdit = {projectId, mode} | null` на уровне списка
  проектов — одновременно открыт ровно один поповер.
- Закрытие: `Escape`, клик по лёгкому прозрачному backdrop (`position:fixed inset:0`, zIndex на
  1 ниже поповера), повторный клик по триггеру. При скролле — закрывать (проще пересчёта coords).
- Триггеры показываются по правам (см. каждый mode). Для `viewer` карточка остаётся, но триггеры
  не кликабельны (без `cursor:pointer`, без `▾`/`+`).
- При открытии полной формы (карандаш) любой открытый quick-edit закрывается.

### Триггеры — это существующие элементы карточки (а не новые кнопки)

1. **Стадия** (`mode:stage`, права owner/admin/editor) — триггер = цветной бейдж стадии
   (добавить `cursor:pointer`, обводку `1px solid rgba(212,175,55,.35)`, значок `▾`). Поповер —
   кастомный **div-список** 5 стадий (НЕ `<select>` — option не берёт inline-стиль): активная
   фон `#d4af37` текст `#1c1c1a`, остальные `#a8a8a3` + цвет-точка стадии, hover `#2a2a2a`.
   Выбор → `updateProject({stage})` с **optimistic-апдейтом бейджа + rollback при ошибке API**.

2. **Платёж** (`mode:payment`, права owner/admin) — триггер = значение «Оплачено» в строке
   финансов (`cursor:pointer`, бейдж «+»). Поповер: `StyledInput type=number` «Сумма ₽»
   (autofocus) + `StyledInput type=date` (default = сегодня) + кнопка «Добавить» (`#d4af37`,
   текст `#1c1c1a`); пустая/нулевая сумма → кнопка disabled. Submit →
   `setProjectPayments([...existingPayments, {paidOn, amount}])` (**replace-all — обязательно
   передавать прежние платежи целиком**); `paid_amount` пересчитает триггер БД — после успеха
   перечитать значение проекта (не считать остаток на клиенте вручную) + toast «Платёж добавлен».

3. **Дедлайн** (`mode:deadline`, права owner/admin/editor) — триггер = «📅 Дедлайн» в строке
   финансов. Поповер: один `StyledInput type=date` (значение = текущий дедлайн). Десктоп —
   `Enter`/`onBlur` сохраняет; мобайл — нативный системный picker (`autoFocus`). Если дата <
   сегодня — некритичное предупреждение «⚠ Дата в прошлом» (`#f8a3a3`, 11px), не блокирует.
   Submit → `updateProject({deadline})`.

4. **Исполнитель** (`mode:executor`, права owner/admin) — триггер = «👤 ФИО» (или «👤 Назначить»
   если пусто). Поповер: `StyledInput` с debounce 300ms → `search_approved_users(query)`; ниже
   список (макс 5, строка 44px: инициалы-аватар + имя). Текущие исполнители показаны сверху с
   крестиком-снятием. Клик по юзеру → `addProjectMember(userId, "editor")` + добавить в
   `executors` jsonb + `updateProject` (executor/executors). Пустой запрос → последние коллеги;
   пусто-результат → «Пользователь не найден». **Согласованность executor(текст)↔executors(jsonb),
   без дублей в members.**

5. **Команда** (`mode:team`, права owner/admin) — НОВОЕ (запрос владельца). Триггер = блок
   команды на карточке: ряд аватаров участников (`UserAvatar`, size ~22) + «+N» при переполнении,
   или «👥 Команда (N)» если участников нет аватарок. Поповер: **переиспользуем существующий
   `MembersManager`** (`projectId, profile, client, showToast, canManage=true`) — список
   участников + роли viewer/editor + поиск-добавление + удаление. Изменения идут через его
   существующие RPC (RLS members write = owner/admin). После изменений — обновить аватары на
   карточке (перечитать members или оптимистично).

### Раскладка поповеров
- **Десктоп:** Portal-поповер `position:fixed`, ширина ~280px (команда — до ~340px), координаты
  от `getBoundingClientRect` триггера, зеркальный сдвиг если у правого края.
- **Мобайл (`useIsMobile`):** тот же Portal как **bottom-sheet** — `position:fixed; bottom:0;
  left:0; right:0; background:#1e1e20; borderRadius:20px 20px 0 0; padding:20px 16px 32px`
  (32px = safe-area), сверху drag-handle (декоративная полоска), backdrop `rgba(0,0,0,.6)`.
  Поля стопкой, строки списков ≥48px. Свайп-закрытие НЕ делаем (только backdrop/крестик/Отмена).
  Поля ввода `fontSize:16` (иначе iOS зумит).

## Часть B — Форма во вкладки (Tab-bar)

Под заголовком Modal — **sticky Tab-bar** (`top:0`, фон `#141414`, `borderBottom:1px solid
#2a2a2e`). 4 вкладки. Активная: `borderBottom:2px solid #d4af37` + текст `#d4af37` +
`fontWeight:600`; неактивная: текст `#6b6b67`, hover white. Под Tab-bar — прокручиваемая зона
(`maxHeight` desktop 60vh / mobile 65vh, `overflowY:auto`). Переключение — мгновенное, **без
перемонтирования Modal** (единый form-state, данные не теряются; меняется только `activeTab`).
Кнопки «Сохранить»/«Создать» и «Отмена» — **всегда внизу Modal**, вне вкладок (не дублировать).

Аккордеон отвергнут сознательно: при 16 блоках всё равно длинный, не даёт ответа «где я».

### Вкладки и состав
1. **📋 Главное** (дефолт, `activeTab='main'`): Название (autofocus) · Заказчик (ClientSelector) ·
   Тип работ · Стадия · Дата начала · Дедлайн · Видимость (Личный/Командный/Маркетплейс) +
   баннер при Маркетплейс. «Паспорт проекта».
2. **💰 Финансы**: Сумма договора · История платежей (таблица дата+сумма + удаление строки) ·
   Добавить платёж · Доли участников (% и ₽, autocomplete, автосумма).
3. **👥 Команда**: Исполнители (autocomplete + внешние по имени, список с удалением) · Команда
   проекта (MembersManager: участники + роли) · Контакты заказчика (телефон/email/telegram).
4. **💬 Детали** (только для существующего проекта; для нового — **disabled** с подписью
   «Доступно после создания», НЕ скрывать — иначе Tab-bar «прыгает»): Ссылки на материалы ·
   Комментарии · Файлы проекта · Задачи проекта · Примечания.

### Ясность (закрывает «пользователь путается»)
- Вкладки — существительные-«куда» с иконками.
- Валидация указывает вкладку: пустое обязательное поле → «Проверьте вкладку Главное:
  не заполнено название» + автопереход на эту вкладку.
- `activeTab` всегда инициализируется `'main'` при открытии формы.
- Навигацию «Далее/Назад» НЕ добавляем (конфликтует со свободными вкладками).

## Мобильность (сводно)
- Карточка: поповеры → bottom-sheet (см. часть A).
- Форма: Modal `maxWidth:100%`, `position:fixed; inset:0; borderRadius:0; overflowY:auto`. Tab-bar
  — 4 кнопки `flex:1`, `fontSize:12`, иконка+слово; если не влезает — `overflowX:auto;
  whiteSpace:nowrap`. Кнопки Сохранить/Отмена — sticky `bottom:0`.
- Все поля `fontSize:16`; тач-зоны ≥44px.

## Вне scope (YAGNI)
- Свайп-закрытие шторок, анимация-физика.
- Глобальный `<style>`-тег ради option-стилей `<select>` (для стадии используем кастомный
  div-список; StyledSelect в форме остаётся с системными option — приемлемо).
- Drag-n-drop участников/ролей, массовые операции.
- Undo стек.

## Риски (проверить при реализации)
1. **Optimistic stage/deadline** — обязателен rollback при reject API (иначе карточка покажет
   неверное состояние).
2. **setProjectPayments = replace-all** — передавать `[...existing, new]`, иначе теряются старые;
   остаток после submit брать из пересчитанного триггером `paid_amount`, не считать на клиенте.
3. **addProjectMember(editor)** — не плодить дубли в `executors` jsonb и members; держать
   `executor`(текст)↔`executors`(jsonb) согласованными (как в текущем saveProject/take_project).
4. **Portal + getBoundingClientRect** — координаты устаревают при скролле/resize: закрывать
   поповер на scroll; чистить listeners в cleanup `useEffect`.
5. **Сброс activeQuickEdit** при размонтировании карточки/перерисовке списка и при открытии
   полной формы — иначе поповер «зависнет» поверх Modal.
6. **Права-гейт** на всех 5 триггерах (viewer — некликабельно; payment/executor/team —
   owner/admin; stage/deadline — owner/admin/editor).
7. **Вкладка «Детали» disabled для нового** — `activeTab` стартует `'main'`, на disabled-вкладку
   попасть нельзя (в т.ч. через валидацию).
8. **Команда на карточке** — после правок в MembersManager-поповере обновлять аватары
   (перечитать members проекта или оптимистично).

## Верификация
- `npm run build` зелёный.
- Каждое из 5 quick-edit действий: меняет данные на проде (живая проверка владельцем);
  optimistic+rollback для stage/deadline; платёж не теряет историю; назначение исполнителя
  кладёт в команду editor; команда меняется из поповера.
- Поповер: не обрезается, не уезжает за край, закрывается на Escape/скролл/клик-вне; на мобайле —
  bottom-sheet.
- Форма: 4 вкладки, переключение без потери данных, «Детали» disabled для нового, валидация
  ведёт на вкладку; контраст активных состояний (тёмный на золоте).
- Права: viewer не видит кликабельных триггеров.
